# Maintainers

## Current Maintainer Set

- `@hassm` (primary maintainer)

## Responsibilities

Maintainers are responsible for:

- reviewing and merging pull requests
- preserving repository quality gates
- handling release execution
- responding to security reports
- maintaining docs and examples accuracy

## Release Authority

Release tags and publish actions are maintainer-owned.
A release requires:

- passing CI
- updated changelog
- version consistency across package files
- completion of `RELEASE_CHECKLIST.md`

## Review Baseline

- prioritize correctness and regression risk
- require tests and docs updates for behavior changes
- reject claims not backed by implementation

## Security Handling

Follow `SECURITY.md` and `docs/security.md`.
Do not request public exploit details in issues.

## Triage Baseline

Use `docs/triage.md` for issue and PR handling flow.
