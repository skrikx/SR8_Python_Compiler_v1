# Support

Use the right channel so requests are handled efficiently.

## Bug Reports

Open a GitHub issue using the bug report template.
Include:

- SR8 version
- Python version
- OS/environment
- exact reproduction steps
- expected vs actual behavior

## Feature Requests

Open a GitHub issue using the feature request template.
Include problem framing, desired outcome, and constraints.

## Security Issues

Do not post exploit details publicly.
Follow [SECURITY.md](SECURITY.md).

## Usage Questions

Open a GitHub issue with:

- command used
- sample input
- observed output
- expected result

## Documentation Improvements

Open a pull request or issue with proposed edits and the command/output that validates the update.
