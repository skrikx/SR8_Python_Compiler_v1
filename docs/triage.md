# Triage

This triage baseline keeps issue and PR handling consistent and lightweight.

## Issue Classification

Use one of:

- bug
- enhancement
- question
- documentation
- security

Reference labels: [labels.md](labels.md)

## Triage Flow

1. Confirm issue is reproducible or clearly actionable.
2. Classify severity and impact.
3. Add labels and milestone (if applicable).
4. Request missing reproduction details when needed.
5. Route security-sensitive reports to private channel.

## PR Review Baseline

- verify scope matches stated objective
- verify tests/docs updated for behavior changes
- verify quality gates pass
- flag regressions and compatibility risks

## Closing Issues

Close when:

- fixed and released
- duplicate of tracked issue
- unsupported or out of scope with rationale
- stale after follow-up and no actionable response
