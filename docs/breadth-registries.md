# Breadth Registries

SR8 formalizes breadth expansion through code-owned registries. These registries describe valid entry modes, artifact families, and delivery targets so new surfaces are discoverable and validated.

## Entry Modes

- file_compile
- api_compile
- frontend_compile
- chat_compile
- intake_resume_compile

## Artifact Families

- canonical_artifact
- promptunit_package
- sr8_prompt
- landing_page_package
- mvp_builder_package
- deep_research_package
- task_graph
- governed_request_package
- multimodal_brief

## Delivery Targets

- json_canonical
- xml_package
- markdown_derivative
- frontend_render
- future_orchestration_handoff

Registries are authoritative and should be consulted for validation, transform compatibility, and front door routing.
